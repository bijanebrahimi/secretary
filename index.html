Secretary

Take the power of Jinja2 templates to OpenOffice and LibreOffice and create reports in your web applications.

Secretary is an alternative solution for creating office documents and reports in OpenDocument Text format from templates that can be visually composed using the OpenOffice.org/LibreOffice Writer word processor.

Secretary use the semantics of jinja2 templates to render ODT files. Most features in jinja can be used into your ODT templates including variable printing, filters and flow control.

Rendered documents are produced in ODT format, and can then be converted to PDF, MS Word or other supported formats using the UNO Bridge or a library like PyODConverter

 

Features

  • WYSIWYG template composing

    Compose templates in Writter. Secretary will keep original formating.

  • No LibreOffice install required

    Secretary works at file level so no binary installation is required.

  • Use familiar template semantics

    Many Python Web frameworks use jinja2 templates or similiars, like Django templates.

  • Use jinja2 filters or create your owns

    Use filters you are already familiar with, or create your owns! Secretay include native support for Markdown throught the markdown filter.

Installing

$ pip install secretary
That's simple!

Alternatively you can also clone the official repo and work with it.

Basic Usage

Secretary implements a class called Render. It takes a single argument called template which can be a filename or a file object. The file most be an ODT document that will be used as template


from secreatary import Render

engine = Render(template)
result = engine.render(foo=foo, bar=bar)

To render a template create an instance of class Render and call the instance's method render passing the template variables as keyword arguments. The function will return the rendered document in binary format.

You can configure the internal jinja2 engine through the environment property of a Render instance. environment is an instance of Environment. This is how you would add a custom filter:


engine = Render(template)

# Configure custom application filters
engine.environment.filters['custom_filer'] = filter_function
result = engine.render(foo=foo, bar=bar)

Composing Templates

Secretary templates are simple ODT documents. You can create them using Writer. An OpenDocument file is basically a ZIP archive containing some XML files.

Template instructions (printing and flow control) should be added through Input Fields. In Writer you insert input fields from Menu > Insert > Fields > Other... or just press CTRL + F2. Then select Functions tab and select Input Field from the list on the left. Finally click Insert button and a dialog will appear where you can type template instructions.

Printing Variables

In the Input Field dialog type {{ variable }} to print te content of variable into the final document.
